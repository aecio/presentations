Busca e análise de dados escalável em tempo-real

por Aécio Santos / @aeciosan

Busca?

  • Busca por termos, intervalos numéricos, datas, etc
  • Busca por relevância
    • TF-IDF
    • Okapi BM25
    • Divergence from Randomness (DFR)
    • Information-based model (IB)
    • etc

Escalável?

  • Indexação e busca é distribuída em nós de um cluster
    • Replicação
    • Sharding

Tempo real?

  • Near realtime
  • Latência pequena do momento de indexação até o documento aparecer nas buscas
  • Consultas são tipicamente repondidas em poucos ms

Análise?

  • Execução consultas ad-hoc aos dados
  • Agregação de dados
  • Visualização de dados agregados

Conceitos básicos

  • Cluster
  • Node
  • Index
  • Type
  • Document
  • Shards/Replicas

API de indexação

	
					PUT /{index}/{type}/{?id}
					

$ curl -XPUT 'http://localhost:9200/twitter/tweet/1' -d '{
    "user" : "kimchy",
    "post_date" : "2009-11-15T14:12:12",
    "message" : "trying out Elasticsearch"
}

API de recuperação de dados

	
					GET /{index}/{type}/{id}
					

curl -XGET 'http://localhost:9200/twitter/tweet/1'
{
    "_index" : "meu_indice",
    "_type" : "meu_tipo",
    "_id" : "1",
    "_version" : 1,
    "found": true,
    "_source" : {
        "empresa" : "kimchy",
        "pessoas" : "2009-11-15T14:12:12",
        "message" : "o rato roeu a roupa"
    }
}

API de Busca

	
					GET /{index}/_search
					
	
					GET /{index}/{type}/_search
					

$ curl -XGET 'http://localhost:9200/twitter/_search?q=user:kimchy'

$ curl -XGET 'http://localhost:9200/twitter/tweet,user/_search?q=user:kimchy'

$ curl - XGET 'http://localhost:9200/_all/tweet/_search?q=tag:wow'

$ curl -XGET 'http://localhost:9200/twitter/tweet/_search' -d '{
    "query" : {
        "term" : { "user" : "kimchy" }
    }
}'
